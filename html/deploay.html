User Manual: Universal MERN Deployment Scripts

This guide explains how to set up and use the deploy.cmd and remote-deploy.sh scripts to deploy multiple MERN stack projects to a single Ubuntu server using only an IP address.

Part 1: One-Time Server Setup

Before you can use the scripts, your Ubuntu server needs to be prepared. You only need to do this setup once per server.

1.1. Connect to Your Server:
Log into your Ubuntu server using SSH.

codeBash
ssh sonusaini@10.0.11.194

1.2. Install Core Software:
Install Node.js, Nginx (web server), and Git.

codeBash
sudo apt update && sudo apt upgrade -y
curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
sudo apt-get install -y nodejs nginx git

1.3. Install and Configure MongoDB:
Install your database.

codeBash
wget -qO - https://www.mongodb.org/static/pgp/server-6.0.asc | sudo apt-key add -
echo "deb [ arch=amd64,arm64 ] https://repo.mongodb.org/apt/ubuntu jammy/mongodb-org/6.0 multiverse" | sudo tee /etc/apt/sources.list.d/mongodb-org-6.0.list
sudo apt update
sudo apt install -y mongodb-org
sudo systemctl start mongod
sudo systemctl enable mongod

1.4. Install PM2 Process Manager:
This will keep your backend applications running.

codeBash
sudo npm install pm2 -g

1.5. Configure Passwordless sudo for Automation (Crucial Step):
This allows the deployment script to configure Nginx without asking for a password.

codeBash
sudo visudo

Scroll to the very bottom of the file and add this line:

codeCode
sonusaini ALL=(ALL) NOPASSWD: /usr/bin/tee, /bin/ln, /usr/sbin/nginx, /bin/systemctl

Save and exit by pressing Ctrl + X, Y, and Enter.

1.6. Create the Master Nginx Configuration File:
This file will hold the configurations for all your projects.

codeBash
sudo nano /etc/nginx/sites-available/master-apps.conf

Paste this exact content into the file:

codeNginx
# Master Nginx configuration for multiple apps on one IP
server {
    listen 80;
    server_name 10.0.11.194;

    # --- LOCATION BLOCKS FOR APPS WILL BE DYNAMICALLY ADDED HERE ---
}

Save and exit. Then, enable this master config:

codeBash
sudo ln -s /etc/nginx/sites-available/master-apps.conf /etc/nginx/sites-enabled/
sudo systemctl restart nginx

Your server is now fully prepared to accept automated deployments.

Part 2: Local PC Setup

On your Windows development machine, you need to prepare the deployment scripts.

2.1. Place the Script Files:
Download or copy the two script files, deploy.cmd and remote-deploy.sh, and place them directly inside the root directory of your MERN project.
Your project structure should look like this:

codeCode
D:\MERN\oms_mern_new_ui\
├── backend/
├── frontend/
├── deploy.cmd             <-- HERE
└── remote-deploy.sh       <-- HERE

2.2. Configure the deploy.cmd Script:
Open deploy.cmd with a text editor (like VS Code or Notepad++). You only need to edit the CONFIGURATION SECTION at the top.

codeBatch
:: #################################################################
:: ###                  CONFIGURATION SECTION                    ###

:: 1. Server SSH Details
SET REMOTE_USER=sonusaini
SET REMOTE_HOST=10.0.11.194

:: 2. Local Backup Drive (Change 'D:' if your backup drive is different)
SET LOCAL_BACKUP_DRIVE=D:

:: 3. Local Project Path (DO NOT CHANGE THIS - It's automatic)
SET LOCAL_PROJECT_PATH=%~dp0

:: 4. Base Production .env File Data
:: UPDATE THESE VALUES with your production database name, secrets, and email credentials.
:: Remember to escape special characters like '|' with a caret '^'.
SET "BASE_ENV_DATA=NODE_ENV=production^
YOUR_MONGODB_CONNECTION_STRING="mongodb://127.0.0.1:27017/oms_database"^
JWT_SECRET=a_very_long_and_secure_production_secret_key^
COOKIE_DOMAIN=%REMOTE_HOST%^
EMAIL_HOST=smtp.gmail.com^
EMAIL_PORT=587^
EMAIL_USER=your-email@gmail.com^
EMAIL_PASS=your16digitgoogleapppassword^
NOTIFICATION_EMAIL_TO=management-email@yourcompany.com"

:: #################################################################

Save the file after making your changes.

2.3. Create the Local Backup Folder:
On your backup drive (e.g., D:), create a folder named backups. The script will automatically create subfolders inside this one.

codeCode
D:\backups\

Your local machine is now ready.

Part 3: How to Deploy

You will use the same deploy.cmd script for both deploying a new project and updating an existing one.

3.1. To Deploy a NEW Project:

  1. Double-click the deploy.cmd file.

  2. The script will ask for a project name. Choose a short, unique, one-word name (e.g., oms-prod).

  3. It will detect that this project does not exist on the server and will run the INITIAL DEPLOYMENT process.

  4. It will then ask for a UNIQUE backend port. Choose a new port that is not in use (e.g., 5001).

  5. It will then ask for backup notes (e.g., Initial deployment of OMS).

  6. The script will run automatically, build the project on the server, configure Nginx, and start the application.

  7. Access your new project at: http://10.0.11.194/oms-prod

3.2. To UPDATE an Existing Project:

  1. Make your code changes on your local machine and save them.

  2. Double-click the deploy.cmd file.

  3. The script will ask for the project name. Enter the name of the project you want to update (e.g., oms-prod).

  4. It will detect that the project already exists on the server and will run the UPDATE process.

  5. It will ask for the EXISTING backend port for that project (e.g., 5001). You must enter the same port you used during the initial setup.

  6. It will ask for backup notes (e.g., Fixed bug in reports section).

  7. The script will run automatically, back up your local code, sync the changes, rebuild the frontend on the server, and restart the backend application. Your changes will be live.
